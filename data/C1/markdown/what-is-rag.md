# RAG 是什麼？

RAG 是 Retrieval-Augmented Generation 的縮寫，中文常翻成「檢索增強生成」。它是一種讓大型語言模型在回答問題前，先從指定資料來源中找出相關內容，再根據找到的內容產生回答的方法。

一般的語言模型只能依靠訓練時學到的知識與使用者輸入回答問題。RAG 則會額外加入一個檢索步驟，讓模型可以參考公司文件、課程講義、產品手冊、FAQ 或知識庫等外部資料。

## RAG 的基本流程

RAG 通常包含以下步驟：

1. 準備文件：收集要讓模型參考的資料，例如 Markdown、PDF、網頁或資料庫內容。
2. 切分文本：把長文件切成較小的段落，避免一次放入太多內容。
3. 建立嵌入：將每個段落轉成向量，方便之後做語意搜尋。
4. 檢索資料：使用者提出問題後，系統會找出最相關的段落。
5. 生成回答：把檢索到的段落與問題一起交給語言模型，要求模型根據上下文回答。

## 為什麼需要 RAG？

RAG 的主要價值是讓模型回答時更貼近指定資料，而不是只靠模型本身的記憶。這對需要使用最新資料、內部文件或專業知識的情境很有幫助。

例如，公司客服系統可以使用 RAG 查詢產品說明書與常見問題，回答使用者關於退貨流程、保固規則或帳號設定的問題。

又例如，課程助教機器人可以使用 RAG 查詢課程講義，回答學生「這一章提到哪些演算法？」或「作業繳交規則是什麼？」等問題。

再例如，企業內部知識庫可以使用 RAG 查詢會議紀錄、技術文件與專案規格，幫助員工快速找到相關資訊。

## RAG 的限制

RAG 並不能保證回答永遠正確。如果檢索到的資料不相關、文件本身過期，或提示詞沒有要求模型嚴格根據上下文回答，模型仍然可能產生錯誤內容。

因此，設計 RAG 系統時需要注意文件品質、切分方式、向量模型、檢索數量與提示詞設計。

## 小結

RAG 可以把語言模型與外部資料連接起來，讓模型回答問題時有更明確的依據。它常用在客服機器人、課程助教、企業知識庫、文件問答與研究資料查詢等場景。
